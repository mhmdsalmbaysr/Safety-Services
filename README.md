# نظام إدارة خدمات السلامة والتراخيص (Safety Services)

نموذج أولي (Prototype) لمنصة **SaaS متعددة المستأجرين** لإدارة عمليات شركات السلامة والتراخيص الفنية،
مبني بـ **Next.js 14 (App Router) + TypeScript + Tailwind CSS** مع دعم كامل للعربية (RTL).

## ✨ الوحدات المنفّذة

| الوحدة | المسار | الوصف |
|--------|--------|-------|
| الصفحة الرئيسية | `/` | عرض تعريفي للمنصة وميزاتها |
| بوابة العميل | `/client` | تقديم طلب ترخيص + محرك التحليل الإجرائي اللحظي |
| النظرة العامة | `/dashboard` | ملخص الأداء وأحدث أرقام الشغل |
| المبيعات | `/dashboard/sales` | قمع المبيعات وعروض الأسعار |
| العمليات والتراخيص | `/dashboard/operations` | معالجة الطلبات ومخرجات محرك التحليل |
| لوحة عزام (التنفيذ) | `/dashboard/execution` | المشتريات، الضريبة الإلزامية، أجور الفنيين، سجل المعوقات وإثبات التوقيع |
| إدارة الاشتراكات | `/dashboard/admin` | لوحة التحكم العليا لإدارة الشركات والباقات |

## 🚀 التشغيل محلياً

```bash
npm install
npm run dev
```

ثم افتح [http://localhost:3000](http://localhost:3000).

## 🗺️ خارطة الطريق (حسب وثيقة SRS)

1. **تأسيس البنية والروابط** — البيئات المستقلة وبوابة العميل ✅ (نموذج أولي)
2. **محرك التحليل والمبيعات** — لوحة العمليات، محرك القواعد، قمع المبيعات ✅ (نموذج أولي)
3. **بوابة التنفيذ والتوثيق** — لوحة عزام، عزل بنود السلامة، التواقيع الميدانية ✅ (نموذج أولي)
4. **إدارة الـ SaaS والتحليلات** — لوحة الاشتراكات والتحليلات ✅ (نموذج أولي)

> جميع البيانات الحالية تجريبية (mock data) لأغراض العرض. الخطوة التالية: ربط قاعدة بيانات
> ومصادقة المستخدمين وتخزين سحابي للملفات.
